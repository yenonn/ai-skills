# Workflow Templates

## Feature Development Patterns

### Full-Stack Feature Template
**Use when**: Developing end-to-end features requiring frontend, backend, and integration

**Team Structure**:
- Frontend Developer (UI/UX focus)
- Backend Developer (API/DB focus) 
- Full-Stack Developer (coordination/integration)
- QA/Tester (validation)

**Timeline**: 1-3 weeks depending on complexity

**Workflow Steps**:
1. **Requirements Analysis** (Day 1)
   - Parse user requirements
   - Define technical scope
   - Identify dependencies

2. **Architecture Design** (Day 1-2)
   - Frontend component structure
   - Backend API design
   - Database schema updates
   - Integration points

3. **Frontend Development** (Day 2-5)
   - Component implementation
   - State management setup
   - UI/UX refinement

4. **Backend Development** (Day 2-5)
   - API endpoint implementation
   - Database changes
   - Business logic
   - Security considerations

5. **Integration & Testing** (Day 5-7)
   - Frontend-backend integration
   - End-to-end testing
   - Performance optimization
   - Bug fixes

6. **Deployment & Validation** (Day 7+)
   - Deployment to staging
   - Production deployment
   - Post-deployment monitoring

**Communication Protocol**:
- Daily sync between frontend/backend
- Integration checkpoint at Day 5
- Final demo and approval at Day 7
